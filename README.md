# 🥗 رشاقة — تطبيق حساب السعرات والتغذية للتخسيس الصحي

تطبيق متكامل **متعدد المستخدمين** لحساب السعرات الحرارية والعناصر الغذائية (الماكروز) بهدف **تخسيس صحي وآمن**.
كل الواجهات **عربية بالكامل** واتجاه **RTL**، والنبرة **مشجّعة وداعمة** بدون أي تأنيب أو تعليق على شكل الجسم.

> ⚠️ **مبدأ أساسي:** الأمان الصحي أولاً. التطبيق يلتزم بحدود سعرات آمنة، ويمنع الأهداف غير الصحية، ويتحوّل تلقائياً لوضع التثبيت عند الوصول للوزن المطلوب.

---

## 🧱 مكوّنات المشروع

| المجلد | الوصف | التقنية |
|--------|-------|---------|
| [`/backend`](backend) | الـ API وقاعدة البيانات ومنطق الحساب | FastAPI · PostgreSQL · SQLAlchemy · Alembic · JWT |
| [`/mobile`](mobile) | تطبيق الموبايل (Android APK) | Flutter |
| [`/web`](web) | لوحة التحكم على الويب | Next.js (App Router) · Tailwind · RTL |

---

## 🎯 الفكرة باختصار

- المستخدم يسجّل بـ **اسم مستخدم وكلمة سر** (مش إيميل)، وكل مستخدم بياناته معزولة تماماً.
- التطبيق يحسب **احتياجك اليومي** بطريقة علمية (Mifflin-St Jeor) ويحدد **هدف سعرات آمن** للتخسيس.
- تسجّل أكلك بطرق كتير: يدوي، من مكتبة أكلات مصرية/سعودية، باركود، تصوير جدول التغذية، أو وصفات.
- تتابع وزنك **بالاتجاه** (مش الرقم اليومي)، ومياهك، ونشاطك، وحالتك المزاجية.
- تقارير أسبوعية وشهرية قابلة للتصدير PDF لمشاركتها مع الطبيب أو أخصائي التغذية.

---

## 🩺 قواعد الأمان الصحي (مدمجة في منطق الحساب)

1. **حساب الاحتياج:** معادلة Mifflin-St Jeor لحساب BMR × معامل النشاط = TDEE.
2. **العجز للتخسيس:** 15–20% من TDEE (أو 500 سعرة، أيهما أقل)، ومعدل نزول أقصاه ~0.5–0.75 كجم/أسبوع.
3. **الحد الأدنى الآمن:** لا يقل الهدف اليومي عن **1200 سعرة للستات** و**1500 سعرة للرجالة**.
4. **منع الأهداف غير الصحية:** لو الوزن المستهدف أقل من النطاق الصحي (BMI < 18.5) يُمنع الحفظ مع اقتراح أقرب وزن صحي.
5. **وضع التثبيت:** عند الوصول للوزن المستهدف يتحوّل تلقائياً لسعرات المحافظة (TDEE بدون عجز).
6. **النبرة الداعمة:** كل الرسائل مشجّعة، ويوم الزيادة عن الهدف تبقى الرسائل داعمة بدون لوم.

## 🛠️ المتطلبات (للتشغيل المحلي)

> هذه الأدوات لازمة لتشغيل المشروع. (في بيئة البناء الحالية بعضها غير مثبّت — راجع `progress.json`.)

- **Python 3.11+** و **PostgreSQL 15+** — للـ backend.
- **Node.js 20+** — للوحة تحكم الويب.
- **Flutter 3.x + Android SDK** — لتطبيق الموبايل.

---

## ▶️ التشغيل المحلي (مختصر — التفاصيل الكاملة تُضاف في المراحل القادمة)

### الـ Backend
<div dir="ltr">

```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # على ويندوز: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env             # واملأ القيم
alembic upgrade head
uvicorn app.main:app --reload
```
</div>

### لوحة التحكم (الويب)
<div dir="ltr">

```bash
cd web
npm install
cp .env.example .env.local
npm run dev
```
</div>

### تطبيق الموبايل
<div dir="ltr">

```bash
cd mobile
flutter pub get
flutter run                      # أو: flutter build apk --release
```
</div>

## 🔐 الخصوصية والأمان

- كلمات السر تُخزَّن **مجزّأة (hashed)** ولا تُحفظ أبداً كنص صريح.
- كل الـ endpoints محمية بـ JWT وتفلتر بـ `user_id` لعزل بيانات كل مستخدم.
- البيانات الصحية حساسة وتُعامَل بسرّية تامة.
